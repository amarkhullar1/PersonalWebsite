Shipping to Legacy Systems: Notes from NHS Spine II

September 5, 2025 · 5 min read

Working on NHS Spine II taught me how to ship features safely into a mission‑critical legacy platform while keeping users happy and ops calm.

Find the seams

Look for integration points where behavior is already modular (e.g., message handlers, adapters). Add capabilities there before touching core workflows.

Safety first

Wrap changes behind feature flags, apply gradual rollout, and build fast rollback paths. For us, end‑to‑end XML tests caught regressions early.

Observability is a feature

Surface metrics and structured logs from day one. Team confidence comes from seeing real usage and error budgets, not just passing tests.

Shared understanding

Legacy systems accumulate context. Pair with long‑timers, document decisions, and run regular demos so everyone stays aligned.

Stack: Python (Flask), Jinja2, HTML/CSS/JS (jQuery), Linux; Jenkins, Splunk; tests with PyTest and Selenium; E2E via XML.
